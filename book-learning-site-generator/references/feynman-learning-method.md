# Feynman Learning Method

## Purpose

Use Feynman learning to help the user actively understand the original book. Do not use it as decorative "explain like I'm five" copy.

## Per-Concept Pattern

For each important concept, generate:

1. Original location: where the book introduces or uses it.
2. Problem: what confusion or task this concept solves.
3. Plain explanation: one paragraph in accessible language.
4. Formal expression: definition, formula, algorithm, or rule.
5. Concrete example: tied to the book's domain.
6. Misconception: a likely wrong interpretation.
7. Teach-back prompt: ask the user to explain it in their own words.
8. Connection: why it matters later in the book.

## Feynman Restatement Area

Each chapter should include a user input area:

- "用自己的话解释本章核心问题"
- "不用公式解释一个关键概念"
- "举一个你自己的例子"
- "写出你仍然卡住的地方"

If persistence is enabled, save this input as chapter memory.

## Self-Test Types

Use lightweight checks:

- Can I explain the concept without copying the book?
- Can I identify what problem it solves?
- Can I connect it to the previous chapter?
- Can I explain the formula symbols?
- Can I give a minimal example?
- Can I spot the common misconception?

## Tone

Use adult learning language:

- clear
- direct
- respectful
- not childish
- not motivational filler

## Avoid

- oversimplifying until the concept becomes false
- replacing original terminology with unrelated metaphors
- introducing external frameworks before the original idea is understood
- creating quizzes that test trivia rather than understanding
